# QTS — 量化交易系统文件系统

> Quantitative Trading System · 基于SMCC五层架构经验的量化交易项目管理体系

---

## 设计原则

1. **五层架构是骨架** — 00-研究 → 10-策略 → 20-回测 → 30-信号 → 40-执行 → 90-复盘，每层职责明确。
2. **数据和策略分离** — 原始数据、中间结果、最终信号分层存储，任何一层出问题不影响其他层。
3. **PROJECT.md 是看板** — 每个策略/项目必须有 PROJECT.md，任何 Agent 读它就能接手。
4. **可回溯** — 每笔交易能追溯到：哪个信号触发 → 哪个策略产生 → 哪份研究支撑。
5. **数据驱动** — 90-复盘追踪实际收益，反向优化策略参数。

## 各层说明

### 00-研究 — 研究成果池

- **定位**：所有策略共享的研究基础，不依附于任何单一策略。
- **宏观**：CPI、PMI、流动性指标、市场风格判断。定期更新。
- **行业**：行业轮动信号、赛道分析、政策影响。
- **个股**：深度研究报告（财报+新闻+技术面），按股票代码分目录。
- **数据源**：API配置、token、本地缓存策略。
- **原则**：研究结论是"原材料"，策略层从中选取自己需要的因子和判断。

### 10-策略 — 策略开发区

- **定位**：每个策略独立目录，包含完整的策略定义和可执行代码。
- **核心文件**：
  - `PROJECT.md` — 策略状态看板（开发中/回测中/运行中/已停用）
  - `策略说明.md` — 策略逻辑、选股条件、因子定义、权重分配
  - `因子库/` — 因子计算代码（PE、ROE、动量、MACD等）
  - `信号生成.py` — 从因子到信号的完整脚本
  - `参数配置.yaml` — 策略参数（阈值、权重、持仓数量等），可版本化
- **原则**：策略代码自包含，可独立运行。参数和逻辑分离。

### 20-回测 — 回测结果

- **定位**：策略的历史验证，按策略+日期隔离。
- **核心产出**：回测报告（收益率、夏普比率、最大回撤、胜率）、交易明细CSV、收益曲线图。
- **原则**：每次回测保存参数快照，确保可复现。同一策略不同参数版本的回测结果可对比。

### 30-信号 — 每日交易信号

- **定位**：各策略每日产出的买卖信号汇总。
- **流程**：各策略信号生成 → 汇总到当日目录 → 综合决策（加仓/减仓/观望）
- **原则**：信号是策略的"产出"，执行层只看信号，不看策略逻辑。

### 40-执行 — 交易执行

- **定位**：信号到订单的转化，持仓管理，风控执行。
- **订单**：每笔订单的完整记录（时间、价格、数量、手续费）。
- **持仓**：当前持仓快照 + 历史持仓。
- **风控**：止损触发、仓位检查、异常告警。
- **原则**：执行层严格按信号操作，不自行判断。风控规则由策略层定义。

### 90-复盘 — 绩效追踪

- **定位**：实际收益追踪，策略优化依据。
- **每日复盘**：当日盈亏、归因分析（哪个策略贡献多少）。
- **月度报告**：累计收益、夏普比率、与基准对比。
- **参数优化**：调参记录（改了什么、效果如何），避免重复调参。
- **原则**：复盘结论反向优化策略，形成"研究→策略→执行→复盘→优化"闭环。

## 典型工作流

```
00-研究（宏观+行业+个股）
    ↓
10-策略（因子计算 + 信号生成脚本）
    ↓
20-回测（历史验证 + 参数调整）
    ↓
30-信号（每日产出买卖信号）
    ↓
40-执行（订单 + 持仓 + 风控）
    ↓
90-复盘（收益归因 + 策略优化）
    ↓
回到 10-策略（优化参数，重新回测）
```

## TradingAgents 市场扫描系统

### 系统架构

```
QTS 项目                          TradingAgents（Docker）
┌─────────────┐                  ┌─────────────────────────┐
│ 00-研究     │ ◄── 调用 ──────► │ /v1/market/insight      │
│ 10-策略     │                  │ 多Agent协同分析          │
│ 20-回测     │                  │ 5个Analyst并行分析       │
│ 30-信号     │                  │ Bull vs Bear 辩论       │
│ 40-执行     │                  │ Research Manager 综合    │
│ 90-复盘     │                  │ 增强版报告（7章节）       │
└─────────────┘                  └─────────────────────────┘
```

### 快速使用

```bash
# 生成市场洞察报告（Markdown）
curl -s http://localhost:8000/v1/market/insight

# 检查容器状态
docker ps --filter "name=tradingagents"

# 启动容器（如果未运行）
docker start tradingagents-scanner
```

### 报告内容（7个标准章节）

| 章节 | 内容 |
|------|------|
| 一、指数数据 | 国内5大指数 + 美股3大指数 + 港股恒生，含收盘点数和涨跌幅 |
| 二、事件联动分析 | 具体事件 + 影响路径 + 因果链条 + 三维度仪表盘（事件驱动/情绪/资金）|
| 三、核心分析 | 多方观点 + 空方观点 + 综合判断 |
| 四、板块建议 | 关注板块 + 回避板块 |
| 五、操作策略 | 仓位建议 |
| 六、风险提示 | 风险因素列表 |
| 七、验证信号 | 后续观察指标 |

### 5个并行分析师

| 分析师 | 职责 |
|--------|------|
| 技术分析师 | MA/RSI/MACD、趋势判断 |
| 宏观分析师 | 板块轮动、资金流向、全球联动 |
| 情绪分析师 | 涨停池、热门股、市场情绪温度 |
| 新闻分析师 | 国内外重要新闻事件影响 |
| 资金分析师 | 主力资金、北向资金、大单异动 |

### 数据源

- A股指数：新浪（akshare）
- 板块数据：同花顺（akshare）
- 全球指数：新浪实时接口
- 新闻数据：财联社、百度经济新闻

```bash
# 命令行使用

```

模板包含的视觉元素：
- 📊 指数数据表格（红涨绿跌）
- 🌡️ 情绪温度仪表盘
- 💰 资金流向仪表盘
- 📊 综合评判框（红=偏多、绿=偏空、灰=中性）
- 🔴 多方观点 / 🟢 空方观点
- 看盘参考（上涨/下跌/横盘确认条件）

## 与SMCC的对比

| 维度 | SMCC（内容创作） | QTS（量化交易） |
|------|-----------------|-----------------|
| 核心循环 | 选题→写作→审核→发布 | 研究→策略→回测→执行→复盘 |
| 原材料 | 00-原料（选题/素材） | 00-研究（宏观/行业/个股） |
| 生产 | 10-主稿（写文章） | 10-策略（写策略代码） |
| 验证 | 老沈审核 | 20-回测（历史验证） |
| 分发 | 20-平台分发 | 30-信号（买卖信号） |
| 执行 | 发布到草稿箱 | 40-执行（下单交易） |
| 复盘 | 90-发布日志（阅读量） | 90-复盘（实际收益） |
| 看板 | PROJECT.md | PROJECT.md |
| 队列 | queue/ | queue/（复用） |
