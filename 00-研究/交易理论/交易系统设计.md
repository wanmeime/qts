## 二、缠论：核心理论基础

### 2.1 技术架构

**缠论 + MACD 辅助判断**

- 缠论提供走势结构和买卖点框架
- MACD 提供趋势确认和背驰判断
- 两者结合使用是缠论完整体系的重要组成部分
- 实际应用中，用缠论做交易系统的人都会重点关注 MACD 指标

### 2.2 缠论的本质

缠论是一个**科学解读市场**的工具，而不是预测工具。

- 对**已生成的走势**形成完整体系
- 提供**可重复验证**的分析框架

### 2.3 缠论核心概念

| 概念 | 说明 |
|------|------|
| **一类买点** | 下跌趋势中的最低点 |
| **二类买点** | 一买确认后的回调低点（右侧交易） |
| **三类买点** | 趋势延续中的加仓点 |
| **一类卖点** | 上涨趋势中的最高点 |
| **二类卖点** | 一卖确认后的反弹高点 |
| **三类卖点** | 趋势延续中的减仓点 |
| **中枢** | 震荡区间，趋势的中继或转折 |
| **底分型** | 三根K线构成，中间K线最低 |

### 2.3 缠论的价值

- **不预测市场**
- 对已发生走势进行**科学归类**
- 提供**完整的买卖点判断框架**
- **可重复验证**，不依赖主观判断

## 三、选股流程

### 3.1 整体流程

```
市场扫描（热门板块 + 领涨股票）
        ↓
缠论分析（买点/卖点/中枢判断）
        ↓
筛选：近期可能产生买点的股票
        ↓
过滤：排除亏损股
        ↓
估值分析（历史位置 + 同业对比）
        ↓
筛选：低估值优先
        ↓
进入选股池
```

### 3.2 第一步：市场扫描

从当前市场中识别：
- **最热门的板块**
- **领涨关注度最高的股票**

### 3.3 第二步：缠论分析

结合缠论对候选股票进行分析：
- 买点判断（一、二、三类买点）
- 卖点判断（一、二、三类卖点）
- 中枢位置分析
- 走势结构判断

### 3.4 第三步：筛选观察

**核心动作**：挑出**有可能在近期产生卖点**的股票

→ 加入自选股，重点观察

## 四、交易原则

### 原则一：亏损股绝对不碰

> 亏损股票绝对不看，不选。

### 原则二：估值分析

重点查看股票的估值数据：
- 处于**历史高位**还是**低位**
- 在**同行业**中属于偏高还是偏低

### 原则三：低估值优先

优先选择**估值较低**的股票进入选股池

## 五、买入方法

### 5.1 多级别联立观察

```
日线级别：出现止跌/买点信号
    ↓
15分钟级别：观察小级别买点
    ↓
分时图：精准捕捉突破时机
```

### 5.2 底分型确认（核心买入信号）

**底分型构成**：三根K线，中间K线最低

**确认条件**：
1. 三根K线形成底分型
2. 第四根K线运行中
3. 第四根K线**突破第三根K线的最高点**
4. 底分型确认 → 买点形成

### 5.3 级别递归

```
日线图：看到底分型
    ↓
15分钟图：确认突破
    ↓
分时图：精准抓突破时机 → 买入
```

### 5.4 倾向的买点

**主要买入：二买（第二类买点）或类二买**

- 一类买点：最低点（左侧交易）
- **二买：右侧交易，更安全**
- **类二买：上涨趋势中最佳买入点（个人修正）**

### 5.5 二买形成逻辑

```
下跌趋势
    ↓
出现一买（最低点）
    ↓
反弹
    ↓
再次下跌，但不跌破一买低点
    ↓
出现有力度的底分型
    ↓
二买即将形成
    ↓
底分型后K线突破高点 → 二买确认
```

### 5.6 类二买（个人修正）

#### 5.6.1 定义

类二买出现在**上涨趋势中的第一个上涨中枢**，形成逻辑与二买相近但不是标准二买。

#### 5.6.2 上涨中枢结构

标准结构：**"上下上"**

#### 5.6.3 类二买的两种形态

**形态一：中枢下沿横盘**
- 第一次向下探底（低点A）
- 第二次向下探底（低点B）
- 低点B ≈ 低点A（基本持平）
- 价格没有继续向下，在原地横盘
- 形成更小级别的中枢震荡

**形态二：中枢上沿横盘**
- 在"上下上"结构的上沿附近
- 横盘形成更小级别的震荡

#### 5.6.4 类二买的确认（买入时机）

```
上涨趋势中的第一个上涨中枢（上下上结构）
    ↓
出现小级别横盘震荡
    ↓
观察更小级别：
  • 底分型出现
  • 或突破小级别中枢上沿（通常是快速突破）
    ↓
突破小级别中枢上沿 → 类二买成立 → 买入
```

#### 5.6.5 买入后的持有观察

```
买入后
    ↓
观察：能否突破大级别上涨中枢的上沿
    ↓
突破后，观察是否"站稳"
    ↓
站稳定义：回调没有再跌入中枢区间
    ↓
回调低点 = 第三类买点（三买）
```

#### 5.6.6 三买的精确定义

> **回调不破前高 = 三买**

突破中枢上沿后，回调的低点没有跌回中枢区间。

#### 5.6.7 类二买的核心特点

| 特点 | 说明 |
|------|------|
| **上涨力度** | 非常大 |
| **上涨速度** | 非常快 |
| **收益区间** | 两个上涨中枢之间的连接线部分 |
| **收益特征** | 速度最快、力度最大的一段 |

#### 5.6.8 类二买的价值

类二买是**上涨趋势中最佳的买入点**：
- 确认后买入
- 享受两个中枢之间最快的上涨段
- 如果能突破中枢上沿并站稳（形成三买），继续持有

## 六、卖出方法

### 6.1 卖出逻辑

**卖出是买入的反向操作**

### 6.2 二卖确认

```
上涨趋势
    ↓
出现一卖（最高点）
    ↓
回调
    ↓
再次上涨，但未突破一卖高点
    ↓
二卖形成
```

### 6.3 卖出信号

- 股票上涨一段时间后出现回调
- 回调结束后再次上涨
- **未能突破前期高点**
- → 二类卖点确认

## 七、仓位管理

### 7.1 基本原则

- **完全基于技术点判断**
- 不过多考虑仓位控制
- 不过多考虑持有时机

### 7.2 资金规模

- 资金量不大
- 在股票中投入的资金占比不高

### 7.3 操作方式

| 情况 | 操作 |
|------|------|
| **常规** | 全仓进，全仓出 |
| **特殊情况** | 涨幅较大且感觉有风险时，减仓一半 |

## 八、止损规则

### 8.1 止损背景

买入后可能失败的情况：
- 突破第三根K线高点后，很快跌回
- 甚至击穿前低，说明一买未形成
- 股票继续下跌

### 8.2 双止损线规则

#### 第一止损线（常规止损）

**触发条件**：
- 二买底分型的**最低点被跌破**
- 跌破后**没有快速收回**
- → 触发止损

#### 第二止损线（强止损）

**触发条件**：
- 买入后股票下跌
- 跌破二买最低点后**继续向下**
- **突破了一买最低点**
- → **无论损失多大，必须止损**

### 8.3 止损逻辑图

```
买入（二买确认）
    ↓
下跌
    ↓
跌破二买底分型低点？
    ├─ 否 → 持有
    └─ 是 → 是否快速收回？
              ├─ 是 → 持有
              └─ 否 → 第一止损线触发 → 止损
                        ↓
              继续下跌，跌破一买低点？
              ├─ 否 → 等待观察
              └─ 是 → 第二止损线触发 → 强制止损
```

## 九、QTS系统目标

### 9.1 系统定位

将个人交易系统和方法论融入QTS，实现**自动化执行**。

### 9.2 完整交易流程

```
┌─────────────────────────────────────────────────────┐
│  1. 市场扫描                                          │
│     → 找出当前市场热点板块                            │
├─────────────────────────────────────────────────────┤
│  2. 选股筛选                                          │
│     → 从热点板块中用方法论筛选值得关注的股票          │
├─────────────────────────────────────────────────────┤
│  3. 自选监控                                          │
│     → 把筛选出的股票放入自选股                        │
│     → 实时监控走势                                    │
├─────────────────────────────────────────────────────┤
│  4. 买入执行                                          │
│     → 出现买入信号时，自动/提醒买入                    │
├─────────────────────────────────────────────────────┤
│  5. 持仓监控                                          │
│     → 监控持仓股票走势                                │
├─────────────────────────────────────────────────────┤
│  6. 卖出执行                                          │
│     → 出现卖出信号时，自动/提醒卖出                    │
├─────────────────────────────────────────────────────┤
│  7. 止损执行                                          │
│     → 触及止损线时，自动/提醒止损                      │
└─────────────────────────────────────────────────────┘
```

### 9.3 系统能力需求

| 能力 | 说明 |
|------|------|
| 市场扫描 | 热门板块、领涨股票识别 |
| 缠论分析 | 买点/卖点/中枢自动识别 |
| 信号生成 | 基于缠论的买卖信号 |
| 自选管理 | 与同花顺自选股同步 |
| 实时监控 | 行情实时推送 |
| 交易执行 | 与券商API对接 |
| 风控执行 | 止损线自动触发 |
