# 缠论 Agent 设计思路（用户口述）

> 以下内容来自用户（缠论实战专家，13年经验）的口述，记录了缠论 Agent 的正确设计逻辑。

---

## 一、核心目标

缠论 Agent 分析一只股票的**核心目标**是：
- **输出操作信号**：买入、卖出、持仓、观望
- **不是预测**，而是基于完全分类给出指导

---

## 二、完全分类的哲学

### 2.1 走势只有三种

缠论的完全分类思想：一只股票的走势无非三种：
- 上涨
- 下跌
- 横盘（盘整）

### 2.2 操作信号对应走势

| 走势 | 操作信号 |
|------|---------|
| 上涨 | 买入/持仓 |
| 下跌 | 卖出/观望 |
| 横盘 | 观望/短线操作 |

### 2.3 核心原则

**不预测，只做完全分类**

当出现什么情况时，我要怎么做？当它出现这个情况的时候，我要怎么做？

---

## 三、分析流程

### 3.1 第一步：看日线级别

1. 看日线图
2. 定位日线级别
3. 用缠论定义判断当前走势

### 3.2 第二步：判断当前笔的状态

在日线级别上，判断当前最新的K线处于：
- **上涨的一笔**中，还是
- **下跌的一笔**中

### 3.3 第三步：判断是否在中枢中

判断当前笔是否处于日线级别的中枢当中：
- 当前笔 + 之前反方向的3笔
- 找这3笔之间**共同重叠的部分**
- 有共同重叠 → 构成中枢
- 无共同重叠 → 不构成中枢

---

## 四、关键概念

### 4.1 笔

- 笔是日线本级别的**最小单位**
- 笔只有两种状态：**上涨一笔**或**下跌一笔**
- 不能随便说"趋势"，只有两个同向中枢才能定义趋势

### 4.2 包含关系处理

**目的**：过滤掉K线重合的部分

| 趋势 | 处理方式 |
|------|---------|
| 上涨趋势 | 取并集（高点取最高，低点取最高） |
| 下跌趋势 | 取交集（高点取最低，低点取最低） |

处理后，只剩下上涨一笔或下跌一笔。

### 4.3 分型

- **顶分型**：中间K线高点最高
- **底分型**：中间K线低点最低
- 顶分型和底分型之间必须有独立K线，才构成有效笔

### 4.4 中继分型 vs 反转分型

| 类型 | 特征 | 含义 |
|------|------|------|
| 中继分型 | 小级别产生中枢，但不构成笔 | 走势继续 |
| 反转分型 | 可能成为顶/底分型 | 走势可能结束 |

**中继分型次数与反转概率**：
- 第1次出现分型：反转概率低
- 2次以上中继分型：反转概率增加
- 第3次分型：反转概率大大增加

### 4.5 中枢判断

**精准定义**：
1. 找方向相反的3笔
2. 找这3笔之间**共同重叠的部分**
3. 有共同重叠 → 构成中枢
4. 无共同重叠 → 不构成中枢

**注意**：方向相反的两笔一定有重叠，但3笔的共同重叠才是中枢。

## 五、报告输出内容

### 5.1 必须输出的内容

1. **当前股价处于什么状态**
   - 日线上涨一笔 / 下跌一笔
   - 是否处于日线级别的中枢当中

2. **中继分型次数**
   - 当前上涨/下跌的一笔已经出现了几次中继分型
   - 这是辅助判断维度，供操作者参考

3. **操作信号**
   - 基于完全分类给出：买入/卖出/持仓/观望

### 5.2 不应该输出的内容

- 不预测是否会反转
- 不预测具体价格目标
- 不给出模棱两可的建议

## 七、与缠论理论的关系

| 缠论概念 | Agent实现 |
|---------|----------|
| 完全分类 | 只输出三种状态，不预测 |
| 不预测 | 只描述当前状态，不猜未来 |
| 级别 | 日线级别为基准 |
| 笔 | 处理包含后的最小单位 |
| 中枢 | 3笔共同重叠区间 |
| 买点/卖点 | 基于笔和中枢的位置判断 |

---

## 六、日线级别完整分析流程

### 6.1 分析步骤

```
1. 判断当前笔的方向（上涨/下跌）
2. 判断当前是否在中枢内/中枢外
3. 判断是否出现顶底分型
4. 判断是否形成买点/卖点
5. 判断买点/卖点类型（1/2/3类）
6. 统计中继分型次数
7. 给出操作信号
```

### 6.2 笔的判断

- 处理包含关系后，只剩上涨/下跌两种状态
- 笔是日线本级别的最小单位

### 6.3 中枢判断

- 方向相反的3笔之间找共同重叠部分
- 有共同重叠 → 构成中枢
- 无共同重叠 → 不构成中枢
- 中枢可无限延伸

### 6.4 趋势判断（基于相邻中枢）

| 中枢关系 | 结论 |
|---------|------|
| 无交叉，A低于A-1 | 下跌趋势 |
| 无交叉，A高于A-1 | 上涨趋势 |
| 有交叉 | 不是趋势，是中枢延伸 |

### 6.5 买点判断（不依赖趋势）

**一买**：当前底分型最低点 < 相邻2-3个底分型的最低点
**二买**：当前底分型最低点 > 前面一笔的最低点
**三买**：当前底分型最低点 > 前面中枢的最高点

### 6.6 买点确认方法

**级别递归法**（精确）：
- 日线：出现底分型
- 15分钟：观察小级别买点
- 分时图：精准抓突破时机

**日线收盘确认法**（备选）：
- 底分型后一根K线收盘价 > 底分型第三根K线最高点

## 七、多级别分析

### 7.1 递归级别

```
日线级别 → 15分钟级别 → 分时图
```

### 7.2 每个级别的分析内容

| 级别 | 分析内容 |
|------|---------|
| 日线 | 笔、中枢、买点/卖点、趋势 |
| 15分钟 | 笔、中枢、买点/卖点 |
| 分时图 | 精确买入时机 |

### 7.3 执行控制

- 信号确认后，**1分钟~半分钟内执行**
- 这是信号到执行的缓冲时间，不是分析级别

## 八、交易规则

### 8.1 买入规则

| 情况 | 操作 |
|------|------|
| 买点出现 | **买入**（只有一种情况） |

### 8.2 卖出规则

| 情况 | 操作 |
|------|------|
| 卖点出现 | **卖出** |
| 止损触发 | **卖出** |

### 8.3 止损规则

**第一止损线（常规）**：
- 当前价格 < 买点底分型最低点
- 且没有快速收回
- → 触发止损

**第二止损线（强止损）**：
- 当前价格 < 一买最低点
- 无论损失多大，必须止损

## 九、设计原则总结

| 原则 | 说明 |
|------|------|
| **完全分类** | 走势只有三种：上涨、下跌、横盘 |
| **不预测** | 只描述当前状态，不猜未来 |
| **基于当下** | 只看当前笔和相邻笔 |
| **级别递归** | 日线→15分钟→分时图 |
| **确定性规则** | 所有判断都有明确的代码实现方式 |

## 十、动态买点确认与止损逻辑（2026-06-08补充）

### 10.1 核心原则

缠论的核心是**不预测**，走势走成什么样就是什么样。

- 买点是在当时发生时确认的
- 如果后续走势打穿了买点，就要**勇于推翻**这个结论
- 打穿了就证明构造失败，需要**触发止损**

### 10.2 买点动态确认

```
当时确认买点（一买/二买）
    ↓
后续走势打穿买点
    ↓
买点构造失败
    ↓
触发止损
```

### 10.3 二买被打穿的逻辑

当二买形成后，如果后续价格打穿了二买：
- 前面的二买构造失败
- 9.17元（二买最低点）被确认
- 出现强势底分型确认
- 9.75元（底分型后第一根K线收盘价）= 精确买入时机

### 10.4 止损触发规则

| 情况 | 操作 |
|------|------|
| 买点被打穿 | 推翻之前的买点结论 |
| 二买被打穿 | 确认新的二买最低点 |
| 触发止损 | 立即止损 |

### 10.5 代码实现要求

- 买点监控：实时监控买点是否被打穿
- 结论推翻：打穿后推翻之前的买点结论
- 止损触发：打穿后触发止损信号
