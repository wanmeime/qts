# 缠论分型

## 定义（第62、65课）

分型是缠论几何结构的最基本元素，由三根相邻的K线组成。

### 顶分型

三根相邻K线中，中间K线的高点是三根中最高的，同时低点也是三根中最高的。

```
    [K2高]
   /      \
[K1高]   [K3高]
  |        |
[K1低]   [K3低]
   \      /
    [K2低]
```

**关键术语**：顶分型的最高点称为该分型的"顶"

### 底分型

三根相邻K线中，中间K线的低点是三根中最低的，同时高点也是三根中最低的。

```
[K1高]   [K3高]
  \      /
   \    /
    [K2高]
   /      \
[K1低]   [K3低]
   \      /
    [K2低]
```

**关键术语**：底分型的最低点称为该分型的"底"

## K线包含关系处理

### 定义

用[di, gi]表示第i根K线的最低和最高构成的区间。当两根相邻K线存在包含关系时（一根K线的高低点全在另一根的范围内），需要进行包含处理。

### 处理规则

**向上时**（gn >= gn-1）：
- 取两根K线的最高点为高点
- 取两根K线低点中的较高者为低点
- 即"取并集"

**向下时**（dn <= dn-1）：
- 取两根K线的最低点为低点
- 取两根K线高点中的较低者为高点
- 即"取交集"

### 顺序原则（结合律）

包含关系不符合传递律，必须按顺序处理：
1. 先处理第1、2根K线的包含关系，得到新K线
2. 用新K线与第3根比较
3. 如有包含关系，继续结合

## 分型的客观性（第65课）

这种定义是唯一的，有统一答案的。即使缠中说禅本人弄错了，也就是错，没有任何含糊的地方。分析的结果与时间无关，与人无关，是客观的，不可更改的。

## 两个同向分型的处理（第65课）

### 两个顶之间没有底

第一种情况：两个顶或底中间有其他的顶和底——这种情况只是把好几笔当成了一笔，继续用一顶一底的原则自然可以解决。

第二种情况：两个顶或底中间没有其他的顶和底——这意味着第一个顶或底后的转折级别太小，不足以构成值得考察的对象，可以忽略其存在。

### 处理原则

对第二种情况（类似对分型中包含关系的处理），可以严格地说：
- **先顶后底**，构成向下一笔
- **先底后顶**，构成向上一笔

所有图形都可以唯一地分解为上下交替的笔的连接。除了第二种情况中的第一个顶或底类似的分型，其他类型的分型都唯一地分别属于相邻的上下两笔。

### 膝盖比喻

用一个最简单的比喻：膝盖就是分型，大腿和小腿就是连接的两笔。

## 代码示例

```python
from enum import Enum

class FractalType(Enum):
    """分型类型"""
    TOP = "top"       # 顶分型
    BOTTOM = "bottom"  # 底分型

def find_fractals(klines):
    """
    分型识别
    
    顶分型：中间K线高点最高，低点也最高
    底分型：中间K线低点最低，高点也最低
    """
    fractals = []
    
    for i in range(1, len(klines) - 1):
        prev = klines[i - 1]
        curr = klines[i]
        next_ = klines[i + 1]
        
        # 顶分型
        if (curr.high > prev.high and curr.high > next_.high and
            curr.low > prev.low and curr.low > next_.low):
            fractals.append({
                'index': i,
                'type': FractalType.TOP,
                'price': curr.high
            })
        
        # 底分型
        if (curr.low < prev.low and curr.low < next_.low and
            curr.high < prev.high and curr.high < next_.high):
            fractals.append({
                'index': i,
                'type': FractalType.BOTTOM,
                'price': curr.low
            })
    
    return fractals
```

## 向上/向下的严格定义（第65课）

假设第n根K线满足与第n+1根的包含关系，而第n根与第n-1根不是包含关系：

- 如果 gn >= gn-1，则称第n-1、n、n+1根K线是**向上**的
- 如果 dn <= dn-1，则称第n-1、n、n+1根K线是**向下**的

gn < gn-1 且 dn > dn-1 的情况不存在（这会违反前面的包含关系假设）。gn >= gn-1 与 dn <= dn-1 不可能同时成立。

---

## K线包含关系的推论（第65课）

1. 用[di, gi]记号第i根K线的最低和最高构成的区间，当向上时，顺次n个包含关系的K线组，等价于[max(di), max(gi)]的区间对应的K线。向下时，顺次n个包含关系的K线组，等价于[min(di), min(gi)]的区间对应的K线。

2. 结合律是最基础的，包含关系不符合传递律（第1、2根包含，第2、3根包含，不意味着第1、3根包含）。必须遵守顺序原则：先用第1、2根K线的包含关系确认新的K线，然后用新的K线去和第三根比。

3. 向上/向下的定义可以严格进行几何定义，不习惯数学符号的人也不难理解。

## 当下性与分型（第65课）

- 如果当下那些K线还没走出来，具体的分型就找不出来，相应的笔、线段、最低级别中枢、高级别走势类型就不可能划分出来
- 一旦当下的K线走出来，就可以当下按客观标准唯一地找出相应的分型结构
- 当下的分析和事后的分析是一样的，分析结果也是一样的

### 显微镜比喻的延伸

- 确定显微镜的倍数，就按看到的K线用定义严格来，没有符合定义的，就是没有
- 30分钟图找不到分型，在年线图里找到分型的机会更小，可能十几年找不到一个也很正常
- 用什么图与以什么级别操作没任何必然关系，用1分钟图也可以找出年线级别的背驰
